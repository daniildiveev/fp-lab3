## Отчет

В лабораторной работе представлена
**стриминговая интерполяция** входных данных (stdin)
по двум алгоритмам — **линейная (отрезками)** и **интерполяция Ньютона**.
Конфигурация задаётся через **CLI**, выход формируется построчно в **stdout**
по мере поступления входных точек.

### Требования (краткая сверка)

1. Алгоритмы: линейная (обязательно), Ньютон по окну `n` ✔
2. Настройки через CLI: выбор алгоритмов,
   шаг дискретизации, формат вывода, режим старта ✔
3. Ввод CSV-подобный (space/tab/semicolon, auto) в stdin,
   **x строго возрастает** ✔
4. Вывод — в stdout, потоково, без ожидания EOF ✔
5. Архитектура: раздельные модули для I/O, алгоритмов,
   генератора сетки, форматирования ✔
6. Идиоматичный Haskell: аппликативный CLI, чистые функции,
   тесты (unit + property) ✔

## Ключевые типы (очень кратко)

- `Point { px :: X, py :: Y }` — входная точка.
- `Window [Point]` — окно из `k` точек для интерполятора.
- `Interval { iLeft, iRight }` — безопасный отрезок для дискретизации.
- `Step`, `StartMode` — шаг и стратегия начала сетки (`align-first | pass-through`).
- `Algorithm = Linear | Newton n` — выбранные алгоритмы.
- `ResultPoint { rpAlgo, rpX, rpY }` — рассчитанная точка на выход.
- Ошибки: `ParseError`, `ValidationError` (в т.ч. нестрого возрастающий `x`), `EngineError`.

## Как оно работает (по конвейеру)

**stdin → Parse → Validate → Engine → Printer → stdout**

1. **Parse**: построчно разбирает строку в `Point` с учётом `--delim auto|space|tab|semicolon`.
2. **Validate**: инкрементальная монотонность `stepValidate` (запрещены `x` дубль/спад).
3. **Engine.Stream**:
   - ведёт по алгоритму своё окно `k` и `ResamplerState` (шаг, старт-режим, курсор);
   - как только окно готово → `safeInterval` → `resample` сетку `x_i` → `evalAt` → `ResultPoint[]`;
   - сливает потоки результатов по возрастанию `x` (при равенстве — по порядку алгоритмов в CLI);
   - на EOF делает `flush` правого края (без дублей благодаря курсору).

4. **Format.Printer**: форматирует строку `linear:`/`newton:` и числа с `--precision`.

## Примеры запуска

**Линейная, шаг 0.7**

```bash
printf "0 0\n1 1\n2 2\n3 3\n" \
| cabal run fp-lab3 -- --linear --step 0.7 --delim space --precision 6
```

**Ньютон, окно n=3, шаг 0.5**

```bash
printf "-1 1\n0 0\n1 1\n2 4\n3 9\n" \
| cabal run fp-lab3 -- -n 3 --step 0.5 --delim space
```

**Оба алгоритма одновременно**

```bash
printf "0 0\n1 1\n2 2\n3 3\n" \
| cabal run fp-lab3 -- --linear -n 3 --step 0.5 --precision 4
```

**Автодетект разделителя**

```bash
printf "0 0\n1;1\n2\t2\n" \
| cabal run fp-lab3 -- --linear --step 0.5 --delim auto
```

**Потоковое поведение (подаём с паузами)**

```bash
{ printf "0 0\n"; sleep 1; printf "1 1\n"; sleep 1; printf "2 2\n"; } \
| cabal run fp-lab3 -- --linear --step 0.7
```
